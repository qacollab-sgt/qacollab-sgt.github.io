An Introduction to QA COLLAB

QA COLLAB is a powerful collaboration platform for Enterprise users to connect, share, manage and extend the use of Test Automation scripts. This platform connects all the stakeholders of the project (Business users, Developers, Non-Technical QA users, Functional Testers, Business Analysts etc.) by sharing QA automation assets. The real benefit of test automation can be reaped with our unique solution called "QA COLLAB".

QA Automation Tools and Framework Support

  • All Open Source Test Automation Frameworks
  • Customized Test Automation Frameworks
  • Commercial Test Automation Tools (UFT One, Katalon, Worksoft, Tosca etc.)
  • Desktop Apps Automation Libraries (AutoIT, PywinAuto, Win32, COM etc.)
  • Web & Mobile Automation Libraries (Selenium, Appium)
  • API Automation Toolsets and Libraries (Commercial & Open Source)
  • Cross Browser Automation Platforms (Browserstack, SauceLabs etc.)

Features

Test Management Tools Integration

Enterprise Test Management Tools like Jira (with Plugins Zephyr scale, Xray etc), ALM, Test Rail etc can be integrated with QA Collab. QA Collab simplifies the integrataion of Test Automation assets, Test Management tools, CI tools, Reporting tools, cloud platforms, ticketing systems and all other QA assets. Users can pull the test cases from Test Management tools to QA Collab, trigger test execution in desired execution platforms (on-prem or cloud) on demand or schedule the test execution and view the test report in desired channel. Test Management tools in the market doesn't have features to integrate QA toolsets as these assets are operating different environment. QA Collab integrates different QA platforms and brings everything under same Medium. The test case execution status and results can also be posted to Test Management tools.

CICD Integration

QA Collab enables the integration of Test Management tools and CI Tools (Jenkins, Bamboo, Azure CI, GitLab CI etc). The Desktop automation tools and libraries in general gives blank screenshot issues, Desktop Graphical content issue (which prevents the tool to interact the application UI) during CICD integration. QA Collab provides the solution for all these issues in CICD execution of Desktop/standalone applications in any VMs/RDCs. When user pulls the tests from Test Management tool and starts or schedules the execution from QA Collab, it triggers to CI jobs in the background and the CI jobs takes care of the test execution in Test Environment.

Seamless Desktop Automation Execution

The automated test execution of Desktop application through CI jobs always been a complicated process. For the desktop automation scripts to run through CI Jobs, the VMs needs to be logged in manually by the user before the execution to enable the graphical content of the Remote Desktops. If this step is not done, the tools wont be able to interact with the Desktop applications UI and eventually the script will fail. QA Collab provides the solution to such issues and it helps for the seamless remote execution for Desktop applications.

Test Execution Environment

Test Execution can be performed in On-Prem or Cloud platforms. The VMs configured in the cloud platforms like Google Cloud, AWS, Microsoft Azure or any other cloud platforms can be used for the Test automation execution and these platforms can be mapped in QA Collab. The cross broswer and mobile automation platforms like Browserstack, SauceLabs also can be integrated with QA Collab for the automated execution. Users can add all execution machines (VMs) details and login details in QA Collab to schedule the test executions in intended machines. The machine will be logged in by QA Collab and thereby no manual login required for CICD executions.

Results Dashboards and Analytics

QA Collab has inbuilt dashboards and analytics to view and analyze the test execution trends and reports. Users can also integrate existing open source or commercial dashboard portals (Power BI, Grafana, Kibana, Reportportal etc) used in the organization with QA Collab platform. The reports and dashboards also can be shared to the users in EMails and also other collaboration channels like MS Teams, Slack etc. The test results can be posted to Test Management tools as well.

Parallel Test Execution

Test Executions can be scheduled to execute the tests in parallel (through multiple VMs) which will reduce the overall test execution time. In addition to this, if the test automation scripts are developed or configured to run in multiple application sessions in parallel, the overall execution time will be reduced 10x time approximately.

Execution Live Preview

Test Automation Executions can be viewed as live video from QA Collab any time during test executions. This live video also can be stored in required storage for the future reference and users can check these vidoes offline. QA Collab also can be integrated with other grid execution and live preview tools like Zalenium.

Cloud based Docker containerized executions

Docker containers on cloud platforms (GCP, AWS, Azure) can be spinned up on user request to perform the test executions (only if the application under test supports the containerized execution). It helps avoid infrastructure cost for test automation execution. It is also useful for parellel test execution which could save time for the overall test execution.
